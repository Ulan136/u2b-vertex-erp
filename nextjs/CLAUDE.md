# U2B Vertex ERP — правила для разработки

## Архитектура и переезд фронта (важно)

Бэкенд — нормальный Next.js API: `src/app/api/v2/*` → controllers → services → repositories → dto. Авторизация и матрица прав enforced на сервере (`withApi` + `apiScreenFor` + `isScreenAllowed`). **Бэкенд переделывать не нужно.**

Основной интерфейс ERP исторически живёт в одном большом статическом файле
`public/sketch_screens.html` (~650 КБ, ~37 экранов, чистый JS). Он **legacy** и
поэтапно переезжает на настоящие Next.js-страницы под маршрутом **`/erp`**
(`src/app/erp/…`), по одному экрану за заход, каждый заход — рабочий деплой.

## ГЛАВНОЕ ПРАВИЛО: новые фичи — только отдельными Next-страницами

- **Любую новую фичу/экран делать как отдельную Next.js-страницу** в
  `src/app/erp/<screen>/page.tsx` с компонентами в `src/components/` и данными
  через существующие `/api/v2/*`. **НЕ добавлять новый функционал в
  `sketch_screens.html`.**
- В `sketch_screens.html` допустимы только **точечные багфиксы** уже
  существующих там экранов, пока они не переехали.
- Общий шелл (сайдбар/шапка/права) — `src/app/erp/layout.tsx` +
  `src/components/erp/*`. Переиспользуемые примитивы — `src/components/ui/*`.
  Клиентские данные — через `src/lib/api.ts` (SWR).
- Права навигации и маршрутов брать из `permissions.dto` (`isScreenAllowed`,
  `visibleScreenKeys`) — единый источник с серверным гейтом.
