## Как работает NCALayer изнутри

NCALayer — это **WebSocket-сервер** запущенный локально на `wss://127.0.0.1:13579`

Любое приложение (браузер, Python-скрипт, наш ERP) может подключиться к нему и отправить JSON-команду:

```json
{
  "module": "kz.gov.pki.knca.commonUtils",
  "method": "signXml",
  "args": [
    "PKCS12",
    "SIGNATURE",
    "<xml данные сертификата поверки>",
    "",
    ""
  ]
}
```

NCALayer сам спросит пользователя: выбрать файл ключа (.p12) и ввести пароль.  
После этого вернёт подписанный XML обратно в наш скрипт.

### Подпись №1 — Поверитель (Ахметов Д.С.)
- Заполняются данные счётчика в форме ЭУПСИ
- Поверитель подписывает своим ЭЦП
- **Это делает наш робот:** заполняет форму + вызывает NCALayer

### Подпись №2 — Руководитель (Молдабаев М.)
- После внесения всех сертификатов за день/партию
- Руководитель подтверждает своим ЭЦП
- **Это тоже делает робот:** batch-подтверждение списком

## Технический стек робота

### Вариант A: Python скрипт (setup.exe / .bat файл)

```python
# robot_ktrm.py
import asyncio
import websockets
import json
import base64
from playwright.async_api import async_playwright

# 1. Данные из нашего ERP (передаются через API)
cert_data = {
    "fio": "Укиева",
    "serial": "42428893",
    "type": "СГВ-15",
    "d1": "09.06.2026",
    "d2": "09.06.2031",
    "stamp": "0752994",
    "water": "г/в",
    "addr": "ул. Казыбек би, 142, кв.59"
}

# 2. Подключение к NCALayer (уже запущен)
async def sign_with_ncalayer(xml_data, p12_password):
    async with websockets.connect("wss://127.0.0.1:13579/", ssl=False) as ws:
        request = {
            "module": "kz.gov.pki.knca.commonUtils",
            "method": "signXml",
            "args": [
                "PKCS12",           # тип хранилища
                "SIGNATURE",        # тип сертификата
                xml_data,           # данные для подписи
                "",
                ""
            ]
        }
        await ws.send(json.dumps(request))
        response = json.loads(await ws.recv())
        return response["responseObject"]  # подписанный XML

# 3. Selenium/Playwright — управление браузером
async def fill_eupsi_form(cert, signed_xml):
    async with async_playwright() as p:
        # Подключаемся к УЖЕ открытому браузеру (не открываем новый)
        browser = await p.chromium.connect_over_cdp("http://localhost:9222")
        page = browser.contexts[0].pages[0]
        
        # Переходим в ЭУПСИ
        await page.goto("https://techreg.gov.kz/eupsi/new")
        await page.wait_for_load_state("networkidle")
        
        # Заполняем форму
        await page.fill('[name="meterType"]',   cert["type"])
        await page.fill('[name="serialNo"]',    cert["serial"])
        await page.fill('[name="checkDate"]',   cert["d1"])
        await page.fill('[name="validDate"]',   cert["d2"])
        await page.fill('[name="stampNo"]',     cert["stamp"])
        await page.fill('[name="address"]',     cert["addr"])
        await page.fill('[name="ownerName"]',   cert["fio"])
        
        # Нажимаем "Подписать" — NCALayer покажет диалог
        await page.click('#btn-sign')
        
        # Ждём завершения
        await page.wait_for_selector('.success-message', timeout=30000)
        print(f"✅ {cert['fio']} · {cert['serial']} — внесено в КТРМ")
```

## Вопрос о пароле ЭЦП — важный момент

> "набрал пароль и поставил подпись"

Здесь два подхода:

### Подход 1: Полуавтоматический (рекомендуем для начала)
- Робот заполняет форму, подходит к моменту подписи
- **Один раз** показывает диалог NCALayer — человек вводит пароль
- Всё остальное автоматически
- **Безопасно** — пароль нигде не хранится

### Подход 2: Полностью автоматический
- Пароль хранится в зашифрованном `.env` файле на компьютере
- Робот сам передаёт пароль в NCALayer через параметры
- **Риск:** если компьютер взломают — ключ скомпрометирован
- НЦА РК **не рекомендует** хранить пароль в открытом виде

### Подход 3: Сессия (лучший баланс)
- Утром поверитель один раз вводит пароль → **сессия открыта**
- В течение дня все подписи идут без повторного ввода
- Аналогично как работает ИС ЭСФ в 1С

## План реализации

### Фаза 1 (1-2 недели): Полуавтоматика
```
Кнопка "🤖 → КТРМ" в ERP
       ↓
Python скрипт запускается
       ↓
Playwright заполняет форму ЭУПСИ
       ↓

       ↓

       ↓
После всех → второй ЭЦП (руководитель)

```

## Вывод

**Это реально.** NCALayer — открытый WebSocket API, к нему можно подключиться из Python.  
Playwright/Selenium — зрелые инструменты для управления браузером.  
Схема "заполнить форму + вызвать подпись + подтвердить второй ЭЦП" — стандартная RPA задача.

**Главное что нужно от вас:**
